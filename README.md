## Описание

**CCAF** (Capellan Confederation Armed Forces)/**ВС КК** (Вооружённые Силы Капелланской Конфедерации) — клиентская часть приложения для работы с каталогом юнитов BattleTech Alpha Strike. Приложение получает данные с FastAPI-сервера `Maskirovka_server` и предоставляет интерфейс для просмотра, фильтрации, сортировки и управления юнитами.

## Функциональность

- 🔍 **Каталог юнитов** с фильтрацией и сортировкой
- 📊 **Детальная карточка юнита** с изображением, характеристиками и расчётом TMM
- ⭐ **Ангар** — локальное сохранение избранных юнитов
- 🛡️ **Управление формациями** по правилам _Battletech: Альфа удар книга командира_
- 🎲 **Трекер партии** — отслеживание брони, структуры, нагрева и критических попаданий
- 💾 **Локальное кэширование изображений** через Cache API
- 📱 **PWA** — поддержка установки на мобильное устройство и офлайн-работа
- 🎨 **Военно-терминальный интерфейс** с тёмной темой и зелёными акцентами

## Стек технологий

- **Vue 3** — современный фреймворк для построения UI
- **Vite** — сборка и dev-сервер
- **Pinia** — глобальное состояние приложения
- **Vue Router** — маршрутизация
- **vite-plugin-pwa** — PWA (service worker, precache)
- **vite-plugin-vue-devtools** — встроенная панель Vue DevTools

## Установка и запуск

### Требования

- Node.js 18+
- Установленные зависимости из `package.json`

### Установка зависимостей

```bash
npm install
```

### Запуск dev-сервера

```bash
npm run dev
```

Приложение будет доступно по адресу: `http://localhost:5173`

Dev-сервер проксирует запросы `/api` на `http://127.0.0.1:8000` (см. `vite.config.js`).

### Сборка production

```bash
npm run build
```

Результат сборки попадёт в каталог `dist/`.

### Запуск production-сборки

```bash
npm run preview
```

### Линтер и форматирование

```bash
# Проверка ESLint
npm run lint

# Автоисправление ошибок ESLint
npm run lint:fix

# Проверка форматирования Prettier
npm run format:check

# Автоформатирование Prettier
npm run format
```

## Настройка адреса сервера

### Dev-режим

В dev-режиме клиент ходит на `/api`, который Vite проксирует на сервер. Адрес прокси можно переопределить переменной окружения:

```bash
# PowerShell
$env:VITE_API_BASE_URL="http://127.0.0.1:20256"; npm run dev
```

### Production

В production используется полный URL сервера, сохранённый в `localStorage` под ключом `ccaf_api_url`. При первом запуске приложение предложит ввести адрес на экране `/setup`.

## Маршруты

| Путь        | Назначение                         |
| ----------- | ---------------------------------- |
| `/setup`    | Первый запуск: ввод адреса сервера |
| `/`         | Каталог юнитов с фильтрами         |
| `/mech/:id` | Детальная карточка юнита           |
| `/force`    | Управление боевыми формациями      |
| `/data`     | Трекер текущей партии              |

## Хранение данных

Приложение использует `localStorage` для локального хранения:

- `ccaf_api_url` — адрес сервера
- `ccaf_hangar` — список ID юнитов в ангаре
- `ccaf_formations` — созданные формации
- `ccaf_active_game` — активная партия в трекере
